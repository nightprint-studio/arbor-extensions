Gitignore Suggester

Scans untracked files in the active repository and proposes well-known .gitignore patterns when it spots common build outputs, IDE metadata, OS junk, or potential secret files.

How it works

  • Every time you open a repo or switch tab, the plugin runs a silent scan. If untracked files match any of its catalog patterns it shows a transient toast with a Review button.
  • The review modal lists each suggestion with a few example paths so you understand why the entry is being suggested. You tick what you want to ignore and submit; the plugin appends the selected lines to .gitignore under a # Added by gitignore-suggester header.
  • Anything you untick is remembered as "dismissed" for that repository — the next scan won't propose it again. Patterns added to .gitignore directly are naturally skipped on the next scan.

Pattern catalog

The plugin recognises common ecosystems out of the box: Rust (target/), Node (node_modules/), Python (__pycache__/, .venv/), .NET / Java (bin/, obj/), IDE caches (.vscode/, .idea/, .vs/), generic build dirs (dist/, build/, out/), bundler caches (.next/, .nuxt/, .svelte-kit/, .turbo/, .gradle/), OS junk (.DS_Store, Thumbs.db, desktop.ini), logs / swap files (*.log, *.swp, *.bak) and environment files (.env, .env.local, .env.*.local).

Manual entry points

Both available from the Command Palette (Ctrl+K):

  • Suggest .gitignore entries — opens the review modal on demand, even if no toast appeared.
  • Reset dismissed gitignore suggestions — clears the per-repo dismissed list and immediately re-opens the modal, handy when you change your mind about an earlier dismissal.
