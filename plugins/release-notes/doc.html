Release Notes

Generates a Markdown release-notes file from a commit range. Does not create a real release on GitHub or GitLab — it only authors the document and lets you save it wherever you want (or pipe it through a downstream release tool).

Wizard

Opens from the Command Palette under Generate release notes…. Two steps:

  1. Range — pick a From ref (default: latest tag) and a To ref (default: HEAD). Both dropdowns mix tags and local branches with a one-letter prefix so they stay easy to scan. Toggle merge commits in or out.
  2. Format — title, grouping (Conventional Commits type, author, or flat), optional Contributors section, and an opt-in ticket enrichment pass.

After the wizard completes the plugin opens Arbor's native save dialog, suggesting a filename derived from the upper bound (release-v1.2.0.md, or a date stamp when targeting HEAD).

Conventional Commits grouping

When grouping by type, summaries are parsed against the type(scope)!: description shape. Recognised types appear in canonical order: Features → Bug Fixes → Performance → Refactoring → Documentation → Styling → Tests → Build → CI → Chores → Reverts. Anything unrecognised lands in an Other bucket — nothing is dropped. The optional scope is rendered as a bold prefix (**auth:** add OAuth login).

Ticket enrichment

When Enrich tickets with Linear / Jira data is on, the plugin scans every commit message for ticket keys matching the configured Lua pattern (default catches the ABC-123 shape used by Linear, Jira and Shortcut). For each unique key it queries arbor.issues.search and appends a sub-bullet with the ticket title, state and link. Lookup failures are silent — enrichment is decoration, not load-bearing.
