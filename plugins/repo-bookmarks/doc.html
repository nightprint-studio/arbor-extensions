Repo Bookmarks

Pin frequently-touched files in the active repository under a custom label and reopen them from the Command Palette without hunting through the file tree.

Workflow

  • Bookmark a file… — opens the file picker rooted at the repo, then asks for a label (pre-filled with the basename so you can usually just hit Enter).
  • Open bookmark… — pick one from a dropdown and hit Open. The file is handed to the OS' default handler via arbor.ui.open_path.
  • Manage bookmarks… — list view with rename (pencil) and delete (trash). Bookmarks whose target no longer exists on disk are tagged missing so cleanup is one click away.

Storage

Bookmarks live in arbor.settings.project under the key bookmarks, JSON-encoded as [{ "label": "…", "path": "/abs/path" }]. Storing them per-repo means the same plugin works across every project without collisions; switching tabs swaps the bookmark set automatically.

Notes

  • Paths are stored absolute, but the picker shows the repo-relative form when applicable for readability.
  • Duplicates are silently rejected: a path is unique within its repo's list.
  • The plugin never modifies the file it points to — opening goes through the system handler (your editor, browser, etc.).
