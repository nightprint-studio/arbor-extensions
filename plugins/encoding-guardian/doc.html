Encoding Guardian

Catches encoding hazards before they reach a commit: UTF-8/CP1252 mojibake (sequences like â€¦ originally …), invalid bytes in the project's declared charset, BOM mismatches, and line-ending mismatches. Each project defines its own encoding contract — Arbor is not all UTF-8.

How it runs

  • on_pre_commit — when this project is opted in, scans the staged set (exactly what git diff --cached --name-only would list) against the project's charset / EOL rules and returns a veto string listing the offenders. The host aborts the commit and surfaces the message. Working-tree garbage you haven't git add-ed is ignored.
  • The hook is opt-in per project. New repos default to disabled — toggle Enable pre-commit encoding check in the gear-icon settings.
  • A global kill switch (Master kill switch) sits above the per-project flag — turning it off disables the hook everywhere.

Command Palette

  • Encoding Guardian: Scan project — manual scan that paints a result modal with the file list and per-file issues.
  • Encoding Guardian: Fix mojibake — auto-repairs every known mojibake sequence (…, —, ', ", •, →, ─) in the active repo. Other issues (BOM, EOL, invalid bytes) need a manual fix or an .editorconfig rule.
  • Encoding Guardian: Open .editorconfig studio — Studio-style modal: a tree of section blocks on the left, a structured form on the right (with a Raw toggle that swaps in a CodeMirror editor for the file verbatim). Creates the file from sensible defaults when it doesn't exist yet.

What gets checked

  • Mojibake — nine known UTF-8/CP1252 sequences plus a catch-all exotic counter for the â€ prefix that indicates corruption even when the third byte isn't one we recognise.
  • Charset — when the project is declared utf-8, any file containing invalid UTF-8 sequences (or a UTF-16 BOM) is flagged.
  • BOM — UTF-8 BOM presence is checked against the project's declared expectation. Disabled by default; enable when the team has a strong "no BOM" or "BOM required" rule.
  • EOL — first 64 KiB of each file is sampled to decide whether it uses lf, crlf, or mixes both. Compared against the project's declared expectation when not set to any.

Per-project encoding

Settings persist under <repo>/.arbor/plugins/encoding-guardian/project.json so each repo carries its own charset / EOL / scope. A legacy CP1252 project can keep its rules without affecting any other repo open in Arbor.

Required permissions

  • fs = "write" — scan reads source files; the fix command rewrites them; the EditorConfig studio writes .editorconfig.
  • git = "read" — resolves the active repo root.

Limitations

  • The scan walks the whole working tree (excluding common build directories — node_modules, target, dist, build, .git, .next, .svelte-kit) and stops at the safety cap (max files, default 5000). It does not currently honour .gitignore beyond those common excludes — declare extra patterns in the Exclude globs setting.
  • Auto-fix only repairs the known mojibake sequences. Exotic â€<X> sequences are surfaced in the scan modal but not auto-rewritten — fix them by hand to avoid guessing.
  • The CodeMirror editor renders .editorconfig with the .properties grammar so keys, values and # comments highlight correctly; [section] headers fall through as plain text (no INI parser ships with the host).
