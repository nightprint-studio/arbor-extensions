Commit Validator

Lints commit messages against the Conventional Commits specification plus a small set of optional rules (ticket reference, subject length, body wrap). In strict mode the plugin blocks the commit before it is created via the on_pre_commit hook; in lenient mode it surfaces the issues as a post-commit warning.

How it runs

  • on_pre_commit — strict mode only. Returns the validation error string and the host aborts the commit.
  • on_commit — runs after every successful commit. In strict mode it shows a quiet success toast; in lenient mode it shows a warning when the message has issues.
  • Right-click any commit in the graph and pick ✓ Validate Commit Message to lint a historical message on demand.
  • Merge commits (Merge …) and revert commits (Revert "…") skip validation.

Required permissions

  • git = "read" — reads commit context from the hook payload.

No network or filesystem access; everything runs locally in Lua.

Default allowed types

feat · fix · docs · style · refactor · perf · test · build · ci · chore · revert

Settings

Open Plugin Manager → gear icon next to Commit Validator. The gear opens the standard plugin-settings panel — a full-height tree-navigated modal whose "Validation" entry groups the rules into General, Conventional Commits, Ticket reference and Length limits cards. All settings are global (apply across every repo).
